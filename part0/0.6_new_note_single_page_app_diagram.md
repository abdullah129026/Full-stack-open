# 0.6: New note in Single page app diagram

Diagram depicting the situation where the user creates a new note using the single-page version of the app.

```mermaid
sequenceDiagram
    participant browser
    participant server

    Note right of browser: User types note content in the text field and clicks the Save button

    browser->>server: POST https://studies.cs.helsinki.fi/exampleapp/notes
    Note right of browser: JavaScript event handler intercepts the form submission
    Note right of browser: Fetch API sends the new note data as JSON
    activate server
    Note left of server: Server processes the new note and saves it to the database
    server-->>browser: JSON response with the new note data
    deactivate server

    Note right of browser: The callback function receives the response with the newly created note
    Note right of browser: JavaScript updates the application state and re-renders the DOM
    Note right of browser: The new note appears on the page without a page reload
```

## Key Points:
- The form submission is handled by a JavaScript event handler (not a traditional form submission)
- Only a single POST request is sent to the server with the note data
- The response is in JSON format, not HTML
- No page reload occurs
- The DOM is updated dynamically by the JavaScript code
- The user experience is much faster compared to the traditional web app
- The server does not need to return a full HTML page
- Multiple notes can be created in rapid succession without any loading delays
